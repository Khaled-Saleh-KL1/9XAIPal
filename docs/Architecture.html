§1

System Topology

9XAIPal is a single-tenant, local-first application. All core components run on the user's local machine — ensuring privacy, low latency, and zero per-token costs.

Frontend
Vite + React (TypeScript) Tailwind CSS · Lucide Icons
Backend
FastAPI (Python 3.11+) SQLAlchemy (Async) · Pydantic v2
Task Queue
Celery + Redis Long-running ingestion and embedding tasks
Database
PostgreSQL with pgvector Relational schema extended with vector-similarity search for RAG
Inference
Ollama — chat & VLM both served by gemma4:26b (one vision-capable model for text + image reasoning) · Embeddings via nomic-embed-text MinerU (Magic-PDF) — high-fidelity PDF-to-Markdown with LaTeX equations & tables · SearXNG — local metasearch for external context

Data Architecture

A relational schema in PostgreSQL extended by pgvector for retrieval-augmented generation.

documents
Root entity for every PDF.
  • idUUIDPrimary key.
  • statusenumqueued · extracting · chunking · embedding · complete · failed
  • reading_orderJSONBLLM-corrected sequence of chunk IDs for multi-column layouts.
  • extractortextWhich engine produced the data (mineru vs pymupdf_fallback).
chunks
The atomic unit of information.
  • sequence_idINTPhysical order in the document.
  • chunk_typeenumtext · heading · table · figure · math · footnote
  • heading_pathTEXT[]Hierarchical path (e.g. ["Introduction", "Related Work"]).
  • markdown / plain_texttextExtracted content.
  • table_jsonJSONBStructured data for tables — LLMs reason about cells.
  • image_refsTEXT[]Links to images stored on disk.
chunk_embeddings
Vectorized representations for similarity search.
  • embeddingvector(768)768-dim vector from nomic-embed-text.
  • embedding_modeltextModel identifier — supports future migration.
section_summaries
Pre-computed hierarchical overviews.
  • levelINT0 = Executive Summary · 1 = H1 · 2 = H2
  • summary_markdowntextLLM-generated section summary.
  • source_chunk_idsUUID[]Chunks used to generate the summary (for citations).
figure_descriptions
VLM-generated technical descriptions of diagrams.
  • description_markdowntextDetailed technical analysis of the image.
  • modeltextVLM used (e.g., gemma4:26b).
§3

The Ingestion Pipeline

Quality-first orchestration — accepts higher ingestion time in exchange for superior retrieval and reading experiences. Driven by app/extraction/pipeline.py.

1

Extraction (MinerU)

subprocess
  • Invokes magic-pdf via subprocess.
  • Performs layout analysis, OCR (if needed), and formula recognition (LaTeX).
2

Structural Chunking

parse
  • Parses content_list.json from MinerU.
  • Falls back to regex-based markdown chunking if MinerU fails.
  • Chunks are split by headings (H1, H2, H3).
3

Asset Management

filesystem
  • Extracted images move to app/storage/images/.
  • Filenames are hashed and linked back to the chunks table via image_refs.
4

Embedding (Async)

celery
  • Dispatches a Celery task to batch-process chunks.
  • Calls Ollama's /api/embeddings endpoint.
  • Results land in chunk_embeddings.
5

Summarization & VLM Analysis

optional
  • Triggers section_summarizer to build the OVERVIEW context.
  • Triggers figure_describer for every chunk of type figure.
§4

Chat & Research Orchestration

The /ask endpoint — driven by app/chat/orchestrator.py — is the most complex part of the system. Every query first passes through intent-based routing.

Intent-Based Routing router.py

ROUTE · 01
LOCAL
Specific to the chunk currently on the user's screen. Includes multimodal support — sends visible images to the VLM.
ROUTE · 02
GLOBAL
Uses pgvector cosine-similarity search across the entire document.
ROUTE · 03
OVERVIEW
Bypasses vector search; uses pre-computed section_summaries for high-level questions ("What is the main contribution?").
ROUTE · 04
EXTERNAL
Triggers SearXNG to fetch web results for out-of-document knowledge.

Research Agent research_agent.py

For complex queries, a "Research Agent" loop is triggered.

01Observe
02Reason
03Act
04Interpret
tool

web_search

Targeted SearXNG queries to surface external evidence.

tool

read_paper_section

Fetches specific chunks by sequence number or heading path.

tool

describe_figure

Uses the VLM to analyze a diagram on the fly.

State: Maintains a temporary research log and synthesizes a final response.

Frontend Architecture

A React application optimized for "Deep Reading."

LibraryView

Polling Real-time updates of ingestion status across the grid/list views.

ReadingView

Granular Reveal Renders units (paragraphs, tables, math) one by one or in small batches — reducing cognitive load.
KaTeX Integration Native rendering of LaTeX math blocks.
Contextual Awareness Tracks which chunk is currently in the viewport to provide "Local" context to the ChatPane.

ChatPane

Citation System Citations render as interactive badges — clicking jumps the ReadingView to the source chunk.
SafeWebImage Wrapper that handles hotlink protection by offering "Open in New Tab" fallbacks for web images.
